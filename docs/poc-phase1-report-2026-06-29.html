§3关键技术决策(避免老板看代码)

决策选了什么为什么
Medium 怎么过反爬 走 Medium 官方 RSS endpoint(medium.com/feed/@user)+ 单独 mediumCrawler + 限速 30 req/min + sameDomainDelay 2s RSS 是 Medium 官方给爬虫的接口 · 不会 403 · 默认走 HTML 抓 0% 成功 → 走 RSS 60-70% 成功
多 token 共一个 URL(binance/Medium 模式) 1-to-N mapping · 一次抓 RSS · 关联回多个 token_id · 输出 N 条 dataset hhwl 数据是 per-token · 不能去重 · 必须每个 token 都有数据传回
有 sitemap 的源怎么抓 用 Crawlee 自带 Sitemap.load() · 取每源前 10 URL · 直接入 generalCrawler 抓 article 页 · 不爬首页 不重复造轮子(项目铁律)· 直接拿 article URL 比从首页爬效率高 N 倍 · 1/3 源能用
per-domain 限速怎么做 用 Crawlee 自带 sameDomainDelaySecs(2024 新增)+ 拆 Crawler · medium 用低速 / general 用高速 不重复造轮子 · Crawlee 没有 per-domain 限速字段 · 但 sameDomainDelay + 拆 Crawler 等价
2c4G 服务器 OOM 怎么解 npm scripts 加 NODE_OPTIONS=--max-old-space-size=3072 + 降 probe 并发到 5 blogpicker 在跑占了 400MB · Node 默认 heap 1.5GB 不够 probe 几百源

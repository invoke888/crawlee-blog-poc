§24 个采集桶 · 怎么抓 · 抽什么字段

桶 1 · 📝 Medium 平台 → RSS endpoint

把 medium.com/@user 转 medium.com/feed/@user · cheerio 解 XML 抽 <item>:title(item/title)· URL(link)· 发布时间(pubDate)· 作者(dc:creator)· 摘要(description / content:encoded 前 280 字)· 分类(category 多个)· guid。1-to-N mapping:多个 token 共一个 publication → 一次抓 RSS 输出 N 条 dataset(每个 token 一份)。

桶 2 · 🗺️ sitemap 自部署站

probe 已探到 sitemap_url → Sitemap.load() 并发拉所有 sitemap → 每源取前 10 URL → 入 generalCrawler 走 default handler 抽 og:title / og:description / og:image / og:site_name / og:type / article:published_time。问题:有些站 sitemap 几万 URL(ASR 26156 · TWLO 22472)· 只取前 10 浪费很多。后续可改成"按 lastmod 排序取最新 50" 或"过滤 /blog/ /news/ 路径"。

桶 3 · 🔍 heuristic 启发式(og 全无)

多重 fallback。标题:og:title → meta[title] → <title> → <h1>。描述:og:description → meta[description] → article p → main p → 任意 p(280 字)。图片:og:image → twitter:image → article img → main img → favicon → 首张 img。发布时间:article:published_time → meta[itemprop=datePublished] → time[datetime] → meta[name=date / publish_date / pubdate]。RSS 自动发现:<link rel=alternate type=application/rss+xml>(本次 0% 命中)。站内 article 链接:<article a> / <main a> / .post a / .entry a 前 5 个。

桶 4 · ⚪ other 标准 og

跟桶 2 同 handler · 直接抓首页 · 走 default handler 抽 og。
